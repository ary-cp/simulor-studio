Welcome to the Simulor developer guides. Simulor is designed to be a local-first, AI-augmented testing studio for modern web applications.

Understanding Scenarios

Scenarios are the core blueprints of Simulor. Instead of testing a single URL, a scenario defines a complete user journey and the load parameters for that journey.

1. Target

The base URL and endpoints you want to probe.

2. Intensity

Concurrency (users) and duration of the test.

3. Actions

Sequence of steps like Login → Post → Feed.

4. Weights

Probability of each action occurring during the run.

Creating Custom Scenarios

Members can create custom scenarios in two ways:

  • Visual Builder: Use the "Builder" tab in the dashboard to generate a scenario without writing code.
  • JSON Configuration: Add a new .json file to the /scenarios directory for advanced control.
{ "name": "custom-smoke-test", "baseUrl": "http://localhost:3000", "users": 20, "duration": 120, "actions": ["login", "search"] }

Installation

Since Simulor is a local-first browser studio, there is no heavy installation required. Just point the agent to your project directory.

npx Simulor-agent@latest ./my-project

Quick Start

To run your first smoke test, simply navigate to the dashboard, enter your localhost URL, and hit "Quick Run".

# Example API Probe curl -X POST http://localhost:4175/api/probe \ -H "Content-Type: application/json" \ -d '{"target": "https://google.com"}'

Target Probes

Probes are the fastest way to check the health of any target (Public URL, Localhost, or File). They return critical metadata like status codes, latency, and page size.

{ "ok": true, "status": 200, "latency": 145, "title": "Example Domain" }

AI Scenarios

Define sequences of user interactions to simulate real-world usage. Scenarios can be configured with user counts, ramp-up times, and custom delays.

  • Light: 5 users, 30s duration
  • Medium: 25 users, 60s duration
  • Heavy: 100 users, 300s duration

Reports & Export

Every test run generates a persistent JSON log and a human-readable report with grade scoring. You can export these as CSV files for stakeholders.
